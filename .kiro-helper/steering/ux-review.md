# UX Review Standards

## Forbidden

- Ugly forms with no spacing
- Spreadsheet-like pages
- Cramped layouts
- Giant tables without pagination
- Wasted whitespace
- Modal overload (more than one modal deep)
- Poor mobile layouts
- Inconsistent cards, colours, typography
- window.prompt() or window.confirm()

## Required Patterns

- Cards for entity summaries
- Tabs for organizing related content
- Timelines for activity/history
- Steppers for multi-step processes
- Pipelines for lifecycle visualization
- Progress indicators for status tracking
- Summary panels for key metrics
- Sticky sidebars for context
- Bulk actions for multi-select operations
- Search with debouncing
- Filters with reset
- Sorting on all columns
- Saved views for filter bookmarks
- Helpful empty states (icon + message + action)
- Loading indicators (spinners in buttons, skeleton rows)
- Confirmation dialogs before destructive actions
- Accessibility (aria labels, keyboard nav, focus management)
- Responsive layouts
- Visual hierarchy (most important info visible first)
- Enterprise polish (consistent spacing, typography, colour)

## Every Screen Must Answer

1. What happened? (recent activity)
2. What is happening? (current status)
3. What requires attention? (alerts, pending items)
4. What should I do next? (clear actions)
